# 404 Page

`sections/404.liquid`

## What it does

Renders the storefront's "Page Not Found" screen — a title and a single
call-to-action button pointing shoppers back to a collection. It's a leaf section
with no blocks and no dynamic data.

## Where it's used

Assigned in `templates/404.json`, which uses the `theme` layout (so it gets the
standard header group, cart/radio sidebars, code-popup, and footer). Preset
category: **Pages**.

## Schema settings

| id | label | type | default |
|---|---|---|---|
| `title-text` | Title Text | text | `Page Not Found` |
| `shop-btn-text` | Shop Button Text | text | `Shop All` |
| `shop-btn-text-color` | Shop Button Text Color | color | `rgba(255,255,255,1)` |
| `shop-btn-background-color` | Shop Button Background Color | color | `#ED1F24` |
| `shop-btn-link` | Shop Button Link | url | `/collections/all` |

(`type: header` divider "Shop Button" omitted — it's an editor label only.)

None.

## Notes

- Purely presentational; all five values are merchant-editable, so don't hardcode
  the title or button copy in the Liquid.
- See [Theme Architecture](architecture.md) for how `templates/404.json` selects
  the `theme` layout.
